When /design is invoked:

1. Ask what needs to be designed — a page, component, user flow, or feature
2. Read the existing frontend structure to align with current patterns
3. Describe the layout, color scheme, typography, and component hierarchy
4. Cover responsive/mobile-first behavior
5. List interactive states (hover, loading, error, empty)
6. If applicable, suggest which existing components to reuse or extend

Always tailor suggestions to a florist commerce context (flowers, bouquets, orders, deliveries).
